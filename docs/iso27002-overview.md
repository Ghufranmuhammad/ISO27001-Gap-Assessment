# ISO/IEC 20022:2022 Overview & Relationship to ISO 27001

## What is ISO 27002?
ISO 27002 is a supplementary standard that acts as a comprehensive reference guide for implementing information security controls. While other standards tell you *what* needs to be secured, ISO 27002 provides the *how*. It offers detailed guidance, best practices, and implementation strategies for the controls listed in Annex A of ISO 27001.

## The Difference Between ISO 27001 and ISO 27002
Understanding the distinction between these two standards is critical for a smooth audit process:

*   **ISO 27001 (The Requirements):** This is the **certifiable** standard. It specifies the requirements for establishing, implementing, maintaining, and continually improving an Information Security Management System (ISMS). If a company says "we are ISO certified," they are certified against ISO 27001.
*   **ISO 27002 (The Guidance):** This standard is **not certifiable**. You cannot get an ISO 27002 certification. Instead, it serves as a toolkit. It expands on the brief control descriptions found in ISO 27001 Annex A, providing the technical and operational meat to help you actually deploy those controls.

## Why ISO 27002 Supports Annex A
Annex A of ISO 27001 is essentially a high-level checklist of 93 controls. On its own, it lacks detail. ISO 27002 maps directly onto Annex A, taking each control and breaking it down into specific implementation guidance, operational considerations, and attributes (such as control types and cybersecurity concepts). It acts as the operational manual for Annex A.

## The 4 Control Themes of ISO 27002
The latest version of ISO 27002 consolidates controls into four distinct, easy-to-navigate structural themes:

1.  **Organizational Controls (5.1 to 5.37):** Focuses on corporate policy, governance, risk management, and operational frameworks (e.g., asset management, return of assets).
2.  **People Controls (6.1 to 6.8):** Deals with human resources, remote working, confidentiality agreements, and security awareness training.
3.  **Physical Controls (7.1 to 7.14):** Covers tangible security measures, including secure areas, entry controls, equipment sitting, and media disposal.
4.  **Technological Controls (8.1 to 8.34):** Focuses on technical security infrastructure, such as encryption, network security, secure coding, and endpoint management.
