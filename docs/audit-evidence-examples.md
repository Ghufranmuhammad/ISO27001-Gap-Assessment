# Audit Evidence Reference Guide

This table provides a mapping of key ISO 27002 controls to concrete, verifiable audit evidence examples that should be maintained for internal and external auditors.

| Control Theme | Control Name | Example Audit Evidence Checklist |
| :--- | :--- | :--- |
| **Organizational** | 5.1 Information Security Policy | Approved policy document with version control, executive sign-off, and intranet read logs. |
| **Organizational** | 5.9 Asset Inventory | Active asset register or CMDB export showing owners, asset classifications, and system criticality. |
| **Technological** | 8.2 Access Control | Semi-annual user access review logs, IAM group permission listings, and change tickets for account creation/revocation. |
| **People** | 6.3 Security Awareness Training | Employee training platform completion records, phishing simulation fail/pass metrics, and onboarding slides. |
| **Physical** | 7.2 Physical Entry Controls | Badge swipe access logs for server rooms, visitor sign-in sheets, and CCTV maintenance records. |
| **Technological** | 8.24 Use of Cryptography | KMS (Key Management Service) policies, encryption standards documentation, and TLS configuration audits for web apps. |
| **Technological** | 8.14 Redundancy (Backup) | Backup schedules, encrypted backup storage configurations, and quarterly backup restoration test results. |
| **Organizational** | 5.24 Information Security Incident Management | Incident log tracker, completed post-mortem/root-cause analysis documents, and evidence of breach notifications if applicable. |
| **Technological** | 8.8 Management of Technical Vulnerabilities | Weekly/monthly network vulnerability scan reports, prioritized patch management logs, and exception approvals. |
| **People** | 6.2 Terms and Conditions of Employment | Signed Non-Disclosure Agreements (NDAs) and employment contracts explicitly stating disciplinary actions for security breaches. |
