# ISO 27001 Gap Assessment Notes

## What is a Gap Assessment?
A gap assessment is a comparison between an organization’s current security posture and a defined standard (ISO 27001).

## What is a “Gap”?
A gap is the difference between:
- What ISO 27001 requires
- What the organization currently has

## Current State vs Desired State
- Current State = existing controls and processes
- Desired State = ISO 27001 compliant ISMS

## Control Mapping
Control mapping links ISO 27001 clauses to actual company controls to check compliance.

## Compliance Maturity
Levels:
- None
- Initial
- Repeatable
- Defined
- Managed
- Optimized

## Why Companies Do This
- To pass ISO certification audits
- To reduce cyber risk
- To meet regulatory requirements
- To improve security posture

## Auditor Usage
Auditors use gap assessments to:
- Identify non-conformities
- Verify control effectiveness
- Recommend corrective actions
